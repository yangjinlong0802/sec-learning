# Day 3 - IP 与 ICMP 抓包分析

## 一、实验目的
掌握 IP、ICMP 报文结构与常见命令（ping、traceroute）。

## 二、实验过程
1. 使用 `ping -c 4 8.8.8.8` 测试网络连通性；
2. 使用 `traceroute www.baidu.com` 追踪路由；
3. 在 Wireshark 中过滤 `icmp`，观察 Echo Request / Reply。

## 三、抓包分析
| 序号 | 源IP | 目的IP | 类型 | 描述 |
|------|------|--------|------|------|
| 1 | 192.168.x.x | 8.8.8.8 | Echo Request | ICMP 请求包 |
| 2 | 8.8.8.8 | 192.168.x.x | Echo Reply | ICMP 应答包 |

## 四、结论
- ICMP 用于网络诊断，不传输用户数据；
- TTL 字段表示经过路由跳数；
- 丢包或高延迟通常意味着网络拥塞或防火墙过滤。
